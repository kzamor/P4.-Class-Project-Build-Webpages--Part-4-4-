TCP Chatroom



My TCP chatroom is a Java-based server that I developed to handle multiple clients simultaneously, ensuring smooth communication between users. I implemented graceful handling of client disconnects and server shutdowns to maintain uninterrupted conversations. The chatroom includes custom features like rolling dice, flipping coins, changing text colors, and muting functions, all designed by me to enhance user engagement. Users can join different "Rooms" within the chatroom, each with its unique set of users and conversations. Messages are sent as payloads, allowing for additional data like usernames and special commands (/roll, /flip) to be processed server-side and broadcasted to all clients. I also implemented basic text formatting features like bold, italics, color changes, and underlining, making conversations lively and expressive without the need for HTML tags. Roll-a-Ball Game



I built my roll-a-ball game by creating a game level with various new World Features and Code Features. In the Unity editor, I made changes like adjusting the RigidBody Component of objects, adding sound effects using the AudioSource component, and customizing the appearance of my Player Game Object. I also delved into coding by adding conditional statements to trigger actions, such as changing the scale of objects based on events or teleporting the player when a specific key is pressed. These modifications were crucial for enhancing the gameplay and creating a more engaging experience for the player. For full credit, I uploaded my project folder to Google Drive, created a WebGL build on itch.io, documented all my modifications with descriptions and code comments, and explained how these changes impacted the mechanics of my game in my submission. Overall, I thoroughly enjoyed experimenting with Unity and C# to create a fun and interactive roll-a-ball game. CLick on the image to play the game yourself!
